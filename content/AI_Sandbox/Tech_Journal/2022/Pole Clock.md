# Pole Clock

<div class="keyword-row">
  <span class="inline-keyword">clock</span>
  <span class="inline-keyword">timezone</span>
  <span class="inline-keyword">ui</span>
  <span class="inline-keyword">design</span>

</div>

Pole Clock is a single dial for multiple time zones, designed for clarity rather than features. It is a small example of tool design that favors simplicity.

I read it as a UI restraint signal. <span class="inline-claim">Good tools often remove options</span>.

<div class="you-might-like">You might like: [[US Senate Votes Daylight Saving Permanent]], [[I Regret My Website Redesign]]</div>

> [!note] Core claim
> Simplicity can be a competitive advantage in utility tools.

> [!question] Reflective question
> Where else could one dial replace five settings?

## signals
- Simplicity can beat feature depth.
- Time tools benefit from visible constraints.
- Multi-timezone work remains a pain point.
- UI clarity drives habit formation.

## my take
The point is restraint. When the task is simple, the tool should be simpler.

<ul class="brain-dump">
  <li><strong>Design:</strong> Constraints reduce errors.</li>
  <li><strong>Signal:</strong> Minimal tools stick.</li>
  <li><strong>Utility:</strong> Clarity beats customization.</li>
  <li><strong>Time:</strong> Coordination needs better defaults.</li>
</ul>
